## 📝 Documentation Maintenance

### Adding New Documentation

1. Create `.md` file in `docs/` directory
2. Add to this index under appropriate category
3. Update [README.md](../README.md) if needed
4. Add internal links to related docs

### Updating Documentation

1. Update the relevant `.md` file
2. Update this index if structure changes
3. Update [README.md](../README.md) links if needed
4. Add "Last Updated" date at bottom of doc

### Documentation Standards

- Use clear, concise language
- Include code examples where applicable
- Add tables for comparisons
- Use consistent heading structure
- Include "Last Updated" date
- Add internal links to related docs
- Use relative links for internal docs (e.g., `[Doc](./file.md)`)
